# Unreal Automation Testing — Rules

## Overview

Unlike Web Backends (Jest), testing in Unreal C++ requires the engine environment to instantiate `UObjects` and parse JSON. We use the **Unreal Automation Framework**.

- **No Play-In-Editor (PIE) reliance for Unit Tests:** Tests should validate data parsing and logic without requiring a map to be loaded.
- **Framework:** `IMPLEMENT_SIMPLE_AUTOMATION_TEST`
- **Location:** All test files must live under `Source/Lunaris/Private/Tests/`.
- **Naming:** Files should be named `[TargetClass]Test.cpp` (e.g., `LunarisJsonParsingTest.cpp`).

---

## 1. What to Test

For the Lunaris Plugin, we focus on **Data Integrity and Network Fallbacks**. We do not test the `FStreamableManager` (Asset Loading) as that requires heavy engine mocking.

| Layer | Mock? | Rationale |
|---|---|---|
| **Core (USTRUCTs)** | **No** | Test `FJsonObjectConverter` directly. Pass a raw JSON string and assert the USTRUCT fields match exactly. |
| **Network (HTTP)** | **Yes** | Use a mock HTTP response (or test the callback logic locally) to assert that a `200 OK` triggers success and a `401` triggers an error delegate. |
| **Runtime (Subsystem)**| **No** | Complex to unit-test. Left for Manual QA or future Integration Tests. |

---

## 2. Implementation Pattern

Use `IMPLEMENT_SIMPLE_AUTOMATION_TEST` for pure logic checks.

- The string parameter `"Lunaris.Unit.Core.JsonParsing"` defines where the test appears in the Unreal Session Frontend UI. Always prefix with `Lunaris.Unit`.
- Use `EAutomationTestFlags::ApplicationContextMask | EAutomationTestFlags::ProductFilter` to ensure the test runs quickly in the Editor.

```cpp
// Source/Lunaris/Private/Tests/LunarisJsonParsingTest.cpp
#include "Misc/AutomationTest.h"
#include "Core/LunarisTypes.h"
#include "JsonObjectConverter.h"

IMPLEMENT_SIMPLE_AUTOMATION_TEST(FLunarisJsonParsingTest, "Lunaris.Unit.Core.JsonParsing", EAutomationTestFlags::ApplicationContextMask | EAutomationTestFlags::ProductFilter)

bool FLunarisJsonParsingTest::RunTest(const FString& Parameters)
{
    // 1. Arrange: Raw JSON exactly as returned by the NestJS Backend
    const FString RawJson = TEXT("{ \"MissionId\": \"qst_01\", \"TargetActor\": { \"ClassPath\": \"/Game/BP_Actor.BP_Actor_C\" } }");

    // 2. Act: Parse using the standard Unreal converter
    FLunarisMissionData MissionData;
    const bool bParsed = FJsonObjectConverter::JsonObjectStringToUStruct(RawJson, &MissionData, 0, 0);

    // 3. Assert: Test Macros
    TestTrue(TEXT("JSON should parse successfully into the USTRUCT"), bParsed);
    TestEqual(TEXT("MissionId must match the JSON field"), MissionData.MissionId, TEXT("qst_01"));
    TestEqual(TEXT("TargetActor ClassPath must match"), MissionData.TargetActor.ClassPath, TEXT("/Game/BP_Actor.BP_Actor_C"));

    return true;
}
```

## 3. Assertions Convention

Unlike Jest (`expect(x).toBe(y)`), Unreal uses `TestTrue`, `TestFalse`, `TestEqual`, and `TestNull`.

- `Always provide a descriptive failure message` as the first argument. If a test fails in the CI, this message is all we have.
  ```cpp
  TestEqual(TEXT("Ensure HTTP status 401 emits an error"), bWasErrorEmitted, true);
  ```

## 4. How to Run the Tests

Tests are not run via CLI by default in development.

1. Open the Unreal Editor.
2. Go to `Tools -> Session Frontend.`
3. Click the `Automation` tab.
4. Check the `Lunaris` folder under `Unit` tests.
5. Click `Start Tests`.

Any code generated by Claude must compile successfully and pass within this framework.
